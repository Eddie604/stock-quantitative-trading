> 本章从零开始，带你理解 AI 如何改变量化交易的游戏规则——以及它为什么不是万能的。

## 6.1 为什么量化交易需要 AI？

### 6.1.1 传统量化的天花板

传统多因子模型的基本框架，在过去 30 年几乎没有本质变化：

```
预期收益 = 因子1 × 暴露度1 + 因子2 × 暴露度2 + ... + 残差
```

这套范式有三个根本性瓶颈：

| 瓶颈 | 解释 | 现实后果 |
|------|------|----------|
| **线性假设** | 世界不是线性的。市盈率 5 倍和 50 倍的含义完全不同，但线性模型只能给它们分配固定的权重 | 错失非线性的交易机会 |
| **边际收益递减** | 第一个发现"低市盈率股票好"的人赚了大钱；第一万个发现的人，超额收益接近零 | 传统因子越来越无效 |
| **市场效率提升** | 高频交易、算法执行、信息传播速度都在加快，简单规律的半衰期不断缩短 | 因子失效速度越来越快 |

用一个直观的例子来说明线性模型的局限：

```
场景：预测一只股票明天是否上涨

传统因子模型看到：
  - 市盈率 = 15      → 给 +0.1 分
  - 动量 = 5%        → 给 +0.3 分
  - 成交量放大       → 给 +0.2 分
  - 总分 = 0.6       → 预测：涨

ML 模型可能学到：
  - 当市盈率在 10~20 之间 AND 动量 > 3% AND 成交量放大 → 涨 (强信号)
  - 当市盈率 < 5 AND 动量 > 3%                     → 跌 (可能是价值陷阱)
  - 当市盈率 > 50 AND 动量 > 5%                     → 涨 (可能是成长股爆发)
  （非线性组合，传统线性模型无法自然学到）
```

### 6.1.2 ML 为量化带来的核心能力

| 能力 | 传统方法 | ML 方法 |
|------|----------|---------|
| 非线性建模 | 手工构造交互项（如 PE × Momentum） | 自动学习任意复杂度的非线性关系 |
| 高维特征交互 | 维数灾难，手搓不过来 | 树模型天然处理，神经网络来者不拒 |
| 在线学习 | 定期重训练（月/季度） | 每天用新数据微调模型 |
| 模式发现 | 人先猜规律，再验证 | 数据驱动，自动发现隐藏模式 |
| 自适应 | 因子失效后手工替换 | 模型自动调整特征权重 |

## 6.2 经典 ML 模型在选股中的应用

### 6.2.1 为什么树模型是主力？

在量化选股这个特定场景，XGBoost 和 LightGBM 的表现**长期碾压**深度神经网络。原因很明确：

| 维度 | 树模型 (XGB/LGB) | 深度神经网络 |
|------|-----------------|-------------|
| **数据类型** | 表格数据。金融数据正是表格数据 | 图像、文本、语音等非结构化数据 |
| **缺失值处理** | 原生支持，自动学习缺失值的最佳分裂方向 | 需要填充，填充方式影响结果 |
| **特征尺度** | 不依赖归一化，完全免疫 | 必须归一化，对异常值敏感 |
| **可解释性** | 特征重要性一目了然 | 黑箱，难以解释 |
| **小样本** | 几千样本就能工作 | 通常需要数万以上 |
| **训练速度** | 快（分钟级） | 慢（小时到天级） |
| **调参难度** | 中等 | 高，超参数空间巨大 |

**核心结论**：在金融表格数据上，树模型 = 更好的性价比。

### 6.2.2 特征工程：从原始数据到模型输入

特征工程是 ML 量化中**最重要、最耗时的环节**。模型只是放大器，特征质量决定上限。

```python
import pandas as pd
import numpy as np

def build_features(df: pd.DataFrame) -> pd.DataFrame:
    """
    从原始价格/成交量/财务数据构建特征矩阵

    Parameters:
    df: 包含 open, high, low, close, volume, 财务字段的DataFrame
        以 (date, stock_code) 为索引
    """
    features = pd.DataFrame(index=df.index)

    # ============ 第一层：价格特征 ============
    close = df['close']

    # 收益率（多周期）
    for period in [1, 3, 5, 10, 20, 60]:
        features[f'return_{period}d'] = close.pct_change(period)

    # 波动率
    for period in [5, 10, 20, 60]:
        features[f'volatility_{period}d'] = (
            df['close'].pct_change().rolling(period).std()
        )

    # ============ 第二层：技术指标特征 ============
    # RSI
    delta = close.diff()
    gain = delta.where(delta > 0, 0).rolling(14).mean()
    loss = (-delta.where(delta < 0, 0)).rolling(14).mean()
    rs = gain / loss
    features['rsi_14'] = 100 - (100 / (1 + rs))

    # MACD
    ema_12 = close.ewm(span=12).mean()
    ema_26 = close.ewm(span=26).mean()
    features['macd'] = ema_12 - ema_26
    features['macd_signal'] = features['macd'].ewm(span=9).mean()
    features['macd_hist'] = features['macd'] - features['macd_signal']

    # 布林带位置
    bb_mid = close.rolling(20).mean()
    bb_std = close.rolling(20).std()
    features['bb_position'] = (close - bb_mid) / (2 * bb_std)

    # ============ 第三层：成交量特征 ============
    features['volume_ratio'] = (
        df['volume'] / df['volume'].rolling(20).mean()
    )
    features['amount_ma_ratio'] = (
        (close * df['volume']) /
        (close * df['volume']).rolling(20).mean()
    )

    # ============ 第四层：横截面排名特征 ============
    # 每天在所有股票中排名（关键！消除市场整体波动的影响）
    for col in [c for c in features.columns if 'return' in c]:
        features[f'{col}_rank'] = features[col].groupby('date').rank(pct=True)

    # ============ 第五层：基本面特征（需财务数据）============
    if 'pe_ttm' in df.columns:
        features['pe_rank'] = df['pe_ttm'].groupby('date').rank(pct=True)
        features['pb_rank'] = df['pb'].groupby('date').rank(pct=True)

    # 注意：所有特征都必须 .shift(1)，防止未来信息泄露！
    features = features.shift(1)

    return features
```

### 6.2.3 标签定义——ML 量化的灵魂

标签定义直接影响模型学到的内容。错误的标签 = 错误的模型。

```python
def build_labels(close_prices: pd.Series,
                 forward_days: int = 5,
                 method: str = 'rank') -> pd.Series:
    """
    构建训练标签

    Parameters:
    close_prices: 收盘价序列
    forward_days: 预测未来多少天的收益
    method:
        'return' - 绝对收益率（简单但不推荐）
        'rank'   - 横截面排名（推荐，更稳定）
        'binary' - 涨跌二分类
    """
    # 未来 N 日收益率
    fwd_return = close_prices.pct_change(forward_days).shift(-forward_days)

    if method == 'return':
        # 原始收益率 —— 牛市全涨、熊市全跌，模型区分度差
        label = fwd_return

    elif method == 'rank':
        # 横截面排名 —— 每天把所有股票的收益率排名，0~1
        # 消除了市场牛熊的影响，模型专注"谁比谁好"
        label = fwd_return.groupby('date').rank(pct=True)

    elif method == 'binary':
        # 二分类 —— 只用排名前 30% 和后 30%
        rank = fwd_return.groupby('date').rank(pct=True)
        label = (rank > 0.7).astype(int)  # 前 30% = 1

    # 去极值 (Winsorize)：把极端值拉到 1%/99% 分位数
    q_low, q_high = label.quantile(0.01), label.quantile(0.99)
    label = label.clip(q_low, q_high)

    return label
```

**为什么推荐排名法 (rank)**：
- 绝对收益率受市场牛熊主导，模型可能只学会了"牛市多买、熊市少买"
- 排名法让模型专注学习"同一市场中，哪些股票更优秀"——这才是 Alpha 的来源

### 6.2.4 时序交叉验证——正确的验证方式

在金融数据上用普通的 K-Fold 交叉验证是**灾难性的错误**。因为金融数据是时间序列，随机打乱会严重泄露未来信息。

```python
from sklearn.model_selection import TimeSeriesSplit

def time_series_cv(X: pd.DataFrame, y: pd.Series,
                   model, n_splits: int = 5):
    """
    时序交叉验证 —— 永远用过去预测未来

    数据集结构：
    [Fold1训练][Fold1测试]
    [Fold1+2训练][Fold2测试]
    [Fold1+2+3训练][Fold3测试]
    ...

    注意：必须按日期划分，不能随机！
    """
    # 获取按时间排序的唯一日期
    dates = X.index.get_level_values('date').unique()
    dates = sorted(dates)

    tscv = TimeSeriesSplit(n_splits=n_splits)
    results = []

    for fold, (train_idx, test_idx) in enumerate(tscv.split(dates)):
        train_dates = dates[train_idx[0]:train_idx[-1]+1]
        test_dates  = dates[test_idx[0]:test_idx[-1]+1]

        X_train = X[X.index.get_level_values('date').isin(train_dates)]
        X_test  = X[X.index.get_level_values('date').isin(test_dates)]
        y_train = y[y.index.get_level_values('date').isin(train_dates)]
        y_test  = y[y.index.get_level_values('date').isin(test_dates)]

        model.fit(X_train, y_train)

        # 评估：用预测排名和实际排名计算 IC
        y_pred = model.predict(X_test)
        ic = np.corrcoef(y_pred, y_test)[0, 1]  # Rank IC

        results.append({
            'fold': fold,
            'train_start': train_dates[0],
            'test_start':  test_dates[0],
            'test_end':    test_dates[-1],
            'rank_ic':     ic,
            'n_train':     len(X_train),
            'n_test':      len(X_test),
        })

        print(f"Fold {fold}: Train [{train_dates[0]} ~ "
              f"{train_dates[-1]}] → Test [{test_dates[0]} ~ "
              f"{test_dates[-1]}] | Rank IC = {ic:.4f}")

    return pd.DataFrame(results)

# 模型训练示例
import lightgbm as lgb

model = lgb.LGBMRegressor(
    n_estimators=200,
    max_depth=6,
    num_leaves=31,
    learning_rate=0.05,
    subsample=0.8,
    colsample_bytree=0.8,
    random_state=42,
    n_jobs=-1,
)

cv_results = time_series_cv(X_features, y_labels, model, n_splits=5)
print(f"\n平均 Rank IC: {cv_results['rank_ic'].mean():.4f}")
print(f"IC 标准差:    {cv_results['rank_ic'].std():.4f}")
print(f"ICIR:         {cv_results['rank_ic'].mean() / cv_results['rank_ic'].std():.4f}")
```

**关键指标——IC 和 ICIR**：

| 指标 | 含义 | 优秀线 |
|------|------|--------|
| Rank IC | 预测排名与真实排名的相关系数 | > 0.03 |
| ICIR | IC 均值 / IC 标准差，衡量预测稳定性 | > 0.5 |
| 月度 IC > 0 比例 | 胜率 | > 60% |

> 对于 A 股，Rank IC 超过 0.05 就已经是相当不错的信号。不要期望看到 0.3 或 0.5——那是未来信息泄露的迹象。

### 6.2.5 ML 量化中的常见陷阱

除了第五章讨论的回测通用陷阱，ML 量化还有一些特有的坑：

| 陷阱 | 描述 | 预防 |
|------|------|------|
| **标签前视偏差** | 标签用到了未来信息（如用 12 月 31 日的年报数据标记 10 月的交易） | 检查每一条数据的确切发布日期 (announcement date) |
| **特征与标签时间错位** | 特征来自 Q1 财报，标签是 Q1 期间的收益——但 Q1 财报 4 月才公布 | 财务数据必须按**公布时间**而非**报告期**对齐 |
| **幸存者偏差的 ML 形态** | 训练数据中只有幸存下来的股票（退市股的数据缺失），模型永远学不到"什么样的股票会退市" | 确保数据集中包含已退市的股票 |
| **过拟合的 ML 形态** | 树深度太大、树太多，完美拟合了历史噪音 | 限制 max_depth (≤6)、使用早停 (early stopping)、控制叶子数量 |
| **分布偏移** | 2019 年的市场规律和 2024 年完全不同，但模型假设分布不变 | 定期重训练、检测数据漂移 |

## 6.3 深度学习在量化中的探索

### 6.3.1 LSTM：时间序列预测的经典方案

LSTM（长短期记忆网络）是处理时序数据最经典的深度学习架构。在量化中，常用于直接预测价格或收益率。

```python
import torch
import torch.nn as nn

class StockLSTM(nn.Module):
    """
    用 LSTM 预测股票未来 N 日收益率
    输入：(batch, seq_len, n_features)
    输出：(batch, 1) —— 预测排名分数
    """
    def __init__(self, input_dim: int, hidden_dim: int = 128,
                 num_layers: int = 2, dropout: float = 0.3):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            dropout=dropout,
            batch_first=True,
            bidirectional=True,  # 双向 LSTM
        )
        self.attention = nn.Sequential(
            nn.Linear(hidden_dim * 2, 1),  # 注意：双向 → hidden_dim*2
            nn.Tanh(),
        )
        self.regressor = nn.Sequential(
            nn.Linear(hidden_dim * 2, 64),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(64, 1),
        )

    def forward(self, x):
        # x: (batch, seq_len=60天, features)
        lstm_out, _ = self.lstm(x)  # (batch, seq_len, hidden*2)

        # 注意力加权
        attn_weights = self.attention(lstm_out)  # (batch, seq_len, 1)
        attn_weights = torch.softmax(attn_weights, dim=1)
        context = (lstm_out * attn_weights).sum(dim=1)  # (batch, hidden*2)

        return self.regressor(context)

# ====== 训练流程 ======
def train_lstm_model(model, train_loader, val_loader, epochs=50):
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)
    criterion = nn.MSELoss()

    for epoch in range(epochs):
        model.train()
        for X_batch, y_batch in train_loader:
            optimizer.zero_grad()
            y_pred = model(X_batch).squeeze()
            loss = criterion(y_pred, y_batch)
            loss.backward()
            optimizer.step()

        # 验证
        model.eval()
        with torch.no_grad():
            val_preds, val_trues = [], []
            for X_val, y_val in val_loader:
                val_preds.append(model(X_val).squeeze())
                val_trues.append(y_val)
            val_ic = np.corrcoef(
                torch.cat(val_preds).numpy(),
                torch.cat(val_trues).numpy()
            )[0, 1]

        print(f"Epoch {epoch:3d} | Val Rank IC: {val_ic:.4f}")
```

### 6.3.2 Transformer / Attention：为什么可能优于 LSTM

| 维度 | LSTM | Transformer |
|------|------|-------------|
| 长距离依赖 | 梯度消失，100 步以上困难 | Self-attention, 理论上无限 |
| 并行计算 | 串行，无法并行 | 完全可并行 |
| 多尺度模式 | 需要堆叠多层 | 多头注意力天然多尺度 |
| 可解释性 | 弱 | Attention 权重可可视化 |
| 数据需求量 | 中 | 大（预训练缓解了这个问题） |

Transformer 在量化中的核心优势是：**市场同时存在日级、周级、月级的模式，多头注意力机制可以同时捕捉这些不同尺度的规律**。

### 6.3.3 GNN：股票关系建模

图神经网络（GNN）用于建模股票之间的关联关系。这在理论上非常吸引人，因为：

- 同一行业的股票有联动
- 供应链上下游互相影响
- 同一指数成分股有资金流动效应
- 股东关系、地域关联都构成图的边

```
图结构示例：

    [贵州茅台] —(同行业)— [五粮液]
         |                     |
    (指数成分)            (指数成分)
         |                     |
    [中国平安] —(同行业)— [招商银行]
         |
    (供应链——保险公司持有银行股)
```

但 GNN 在实际应用中面临巨大挑战：

- 图的动态性：股票关系每天都在变
- 图的规模：5000 只股票的全连接图计算量巨大
- 冷启动问题：新上市的股票没有历史关系数据

### 6.3.4 深度学习的现实困境——坦诚版

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| **数据量太少** | 致命 | 5000 只股票 × 20 年 = 约 2500 万条日线数据。听起来很多，但 OpenAI 用 1 万亿 token 训练 GPT。量化是 DL 领域的"小数据"场景 |
| **信噪比极低** | 致命 | 股价 90%+ 的波动是噪声。DL 擅长在海量高质量数据中找模式，不擅长在噪声中大海捞针 |
| **可解释性差** | 高 | 银行、基金的金工部门不敢用解释不了的模型。亏了钱说不清为什么 = 丢工作 |
| **调参地狱** | 高 | 架构 × 超参数 × 训练策略的组合空间巨大，容易变成"花式过拟合" |
| **硬件成本** | 中 | 个人一台 GPU 能做的事有限 |
| **复现困难** | 高 | 很多论文的结果无法复现——也许作者自己也复现不了 |

> **诚实地说**：对个人开发者而言，**XGBoost/LightGBM + 好的特征工程** 在绝大多数情况下优于 LSTM/Transformer。深度学习在量化中的价值更多在于前沿探索和学术研究，而非实用策略。

## 6.4 大语言模型 (LLM)——量化新范式

### 6.4.1 LLM 能做什么？

2024 年以来，LLM 为量化交易打开了一个全新的 Alpha 来源：**非结构化文本数据**。

```
传统量化处理的数据：           LLM 新增的数据维度：

┌─────────────┐              ┌─────────────────────┐
```

**LLM 在量化中的 5 个实际应用：**

| 应用场景 | 成熟度 | 说明 |
|----------|--------|------|
| **财报情感分析** | 高 | 分析管理层讨论的语气和措辞变化，比财务数字本身更早反映问题 |
| **新闻事件分类** | 高 | 将新闻自动分类为利好/利空/中性，并识别对具体股票的影响 |
| **研究摘要生成** | 中 | 每天读 500 份研报，LLM 用 5 分钟总结核心观点 |
| **交易日志分析** | 中 | 分析交易记录，自动发现"我今天为什么亏了" |
| **代码辅助** | 高 | 写回测框架、调 Bug、解释别人的策略代码 |

### 6.4.2 LLM 不能做什么？

> **重要警告：LLM 不能直接预测股票价格！**

不要问 ChatGPT 或 Claude 这样的问题：
- "明天贵州茅台涨还是跌？"
- "推荐 3 只下周会涨的股票"
- "预测上证指数下周点位"

这会得到以下几种结果：
- **幻觉**：编一个看似合理实则完全虚构的答案
- **免责声明**："我无法预测股票价格，投资有风险……"
- **看似有理的分析**：基于它训练数据中的通用知识，不包含任何你需要的实时信息
- **最坏情况**：给你一个听起来很专业的错误预测，你信了，然后亏钱

LLM 不是水晶球。它的本质是**语言理解和生成引擎**，不是**股价预测引擎**。

### 6.4.3 LLM 情感因子：一个新的 Alpha 来源

虽然 LLM 不能直接预测股价，但它可以构建**情感因子**——一种传统量化方法难以获取的 Alpha 信号。

```python
"""
LLM 情感因子构建流程

这个示例展示了如何用 LLM 从财经新闻中提取情感信号
注意：这只是一个概念原型，生产环境需大量优化
"""

from openai import OpenAI

def extract_sentiment_factor(news_text: str,
                              stock_name: str) -> dict:
    """
    用 LLM 从新闻文本中提取结构化情感信号

    Returns:
        {
            'sentiment': 1.0 ~ -1.0,  # 正向/负向情感强度
            'relevance': 0.0 ~ 1.0,    # 与目标股票的相关性
            'category': str,           # 事件类型
            'urgency': 0.0 ~ 1.0,      # 紧迫程度
            'summary': str,            # 一句话摘要
        }
    """
    client = OpenAI()

    prompt = f"""分析以下财经新闻对【{stock_name}】的影响。

请以 JSON 格式回复，字段包括：
- sentiment: 情感分数，-1.0（极度负面）到 1.0（极度正面）
- relevance: 与【{stock_name}】的相关度，0.0（无关）到 1.0（直接相关）
- category: 事件类别（业绩/政策/行业/公司/市场/其他）
- urgency: 可能引发短期波动的程度，0.0 到 1.0
- summary: 一句话摘要

新闻内容：
{news_text[:3000]}  # 截断以控制 token 消耗
"""

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
        response_format={"type": "json_object"},
        temperature=0.0,  # 分析任务用 0，不要创造性
    )

    return json.loads(response.choices[0].message.content)

# 因子构建逻辑
# 每天都对所有新闻做分析 → 聚合股票级别的日度情感分数
# → 这是一个新的因子列 → 输入到 LightGBM 模型
```

**情感因子的实证潜力**：

| 研究来源 | 结论 |
|----------|------|
| 学术文献 | 新闻情感因子的 Rank IC 通常在 0.02~0.05，属于中等偏弱的 Alpha 但与其他因子低相关 |
| 业界实践 | 头部对冲基金已广泛使用 NLP 处理新闻（但用的是定制模型，不是通用 LLM） |
| 个人开发 | 成本是主要瓶颈——每天分析 5000 只股票的所有新闻，API 调用费用可达数百元/天 |

### 6.4.4 多模态：从 PDF 到结构化数据

量化数据不会整洁地摆在你面前等着用。LLM 的多模态能力让"脏数据清洗"变得可行：

```
数据形态                    LLM 处理

PDF 财报 ──────────────→ 提取关键数字 + 文本情感
扫描件 PDF (OCR) ──────→ 数字化 + 结构化
电话会议录音 ──────────→ 语音转文字 → 情感分析
会议纪要图片 ──────────→ OCR → 关键信息提取
非标格式表格 ──────────→ 解析为标准 CSV
政府公文 PDF ──────────→ 政策影响分类
```

### 6.4.5 LLM 在量化中的局限性——必须诚实的部分

| 局限 | 严重程度 | 说明 |
|------|----------|------|
| **幻觉 (Hallucination)** | **致命** | 金融对错误零容忍。LLM 可能凭空编造一个"业绩预增 50%"然后你据此交易——这会导致真实亏损 |
| **时效性** | 高 | 模型知识有截止日期，新的市场动态它不知道 |
| **成本** | 高 | GPT-4 API 约 $0.03/1K tokens。每天分析 10000 条新闻，月费轻松过万元 |
| **速度** | 高 | API 调用延迟 0.5~3 秒。做日内高频？别想了 |
| **不可复现** | 中 | 同样的 prompt，不同时间可能得到不同的回答（即使 temperature=0，云端的微小变化也可能影响） |
| **合规风险** | 中 | 用 LLM 做投资决策，出了问题谁来负责？监管对此尚未明确 |
